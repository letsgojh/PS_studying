# 말이 되고픈 원숭이(골드3)
## 코드 설명

1. visit : 3차원 배열 [y좌표][x좌표][말이동횟수] -> 모두 MAX_VALUE로 초기화
2. queue에 1차원 배열로 [y좌표][x좌표][그냥이동횟수][말이동횟수] 넣어줄 것임. 가장 첫번째로는 [1][1][0][0] 추가
3. 만약 말 이동 횟수가 K 보다 적다면 말 이동 경우의 수 탐색
4. 그냥 이동 경우의 수 탐색
5. 탐색 할 때 visit의 값보다 "총 이동 횟수"가 적은 경우에만 queue에 추가해줌
6. q가 빌 때까지 탐색

# 애로사항
- visit 배열을 처음에 2차원 배열로 사용하고 '말 이동횟수'는 고려하지 않았음
- 2차원 배열로 쓸 경우 그 위치가 '말 이동'해서 크랙 플레이로 레전드 무빙을 찍을 수 있는 위치임에도 불구하고 그냥 넘어가는 경우 발생
