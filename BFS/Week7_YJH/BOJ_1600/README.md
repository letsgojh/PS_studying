# 1600 : 말이 되고픈 원숭이
## 코드 설명

1. 3차원 배열을 만들어, 3차원 index를 말처럼 행동하는 횟수로 보았다. ex) distance[3][4][5] = 말처럼 3번 이동
2. 먼저 말처럼 8방향을 이동한 뒤, 일반적으로 4방향을 움직이는 로직으로 구성
3. 3차원 index값이 k보다 작은 경우 계속 말처럼 움직이는 것을 허용
4. distnace[][H-1][W-1] 순회를 k번만큼 돌면서 가장 작은 값을 출력한다.

## 애로 사항
1. 초반 말 움직임을 잘못설정하여. 최소 3시간은 해맸다. 분명 맞는 로직인데, Queue에 계속 들어가지 않아서 적잖이 당황했다; horse_y[8] = {-1,-2,-2,-1,2,1,1,2} 로 구성했었다;
2. 이전에 풀어봤던 벽부수고 움직이기와 비슷한 문제인 줄 알았는데, 3차원 index를 정하는 부분이 달랐다.
3. 또한 초반에는 2차원으로 구성 후, horse_move_check를 사용하여 움직임을 다 사용하는지만 체크하였는데.. 메모리 초과가 났다.(Queue 폭발)
